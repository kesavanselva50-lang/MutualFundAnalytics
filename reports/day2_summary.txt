Day 2 Report – Data Cleaning & SQL Database Design

Objective

To clean and validate mutual fund datasets, design a SQLite database schema, load cleaned data into the database, and prepare analytical SQL queries for reporting.

Work Completed

1. Data Cleaning

Processed and cleaned 10 datasets:

1. Fund Master
2. NAV History
3. AUM by Fund House
4. Monthly SIP Inflows
5. Category Inflows
6. Industry Folio Count
7. Scheme Performance
8. Investor Transactions
9. Portfolio Holdings
10. Benchmark Indices

2. Data Validation

* Converted date columns to proper datetime format.
* Removed duplicate records.
* Forward-filled missing NAV values where applicable.
* Validated NAV values greater than zero.
* Standardized transaction types (SIP, Lumpsum, Redemption).
* Validated transaction amounts greater than zero.
* Verified KYC status values.
* Checked scheme performance metrics and expense ratio ranges.
* Validated portfolio holding weights and benchmark values.

3. Processed Data Generation

Generated 10 cleaned CSV files and stored them in the `data/processed` directory for downstream analytics and database loading.

4. Database Design

Designed a SQLite star schema consisting of:

* dim_fund
* dim_date
* fact_nav
* fact_transactions
* fact_performance
* fact_aum

Defined primary keys and foreign key relationships for analytical reporting.

5. Database Loading

* Connected SQLite using SQLAlchemy.
* Loaded cleaned datasets into the `bluestock_mf.db` database.
* Verified successful loading using row count validation scripts.

6. SQL Analytics

Created 10 analytical SQL queries covering:

* Top funds by NAV
* Average NAV analysis
* Transaction analysis
* State-wise transaction distribution
* Expense ratio analysis
* Fund performance analysis
* AUM analysis
* Category inflow analysis
* Portfolio holdings analysis
* Benchmark index analysis

7. Documentation

Prepared:

* schema.sql
* queries.sql
* data_dictionary.md

Documented dataset structures, column definitions, and business meanings.

Deliverables Completed

* 10 Cleaned CSV Files
* SQLite Database (`bluestock_mf.db`)
* Data Cleaning Script (`data_cleaning.py`)
* Database Loading Script (`load_to_db.py`)
* Verification Script (`verify_db.py`)
* Database Schema (`schema.sql`)
* SQL Queries (`queries.sql`)
* Data Dictionary (`data_dictionary.md`)

Outcome

Successfully completed data cleaning, database design, data loading, and SQL analytics preparation for the Mutual Fund Analytics Capstone Project. The project is now ready for advanced SQL analysis, reporting, and dashboard development in the next phase.
